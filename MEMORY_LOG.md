# USER_HISTORY_TRACKING
- **Purpose:** Store a rolling 30-day summary of user interactions to maintain long-term context, understand user pain points, and provide personalized guidance.
- **Structure:** Each entry should include date, user ID, key query, and the resolution/outcome.
- **Retention:** Review and prune entries older than 30 days during heartbeats.
- **Goal:** Identify repeating struggles (e.g., users always getting stuck at Step 6 of eTAX) to proactively improve guidance.
